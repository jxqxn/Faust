# AI 过度系统化失败记录

> 目的：记录 AI 在叙事/设计推理中反复出现的"过度系统化"失败模式，作为后续会话的防坑参考。
>
> 关联文档：`narrative-card-game-research.md`（研究主体）、`original-pitch.md §3.5`（断言分类纪律）、HTML demo 的 v1→v5 track record（复刻理解层的 track record）。
>
> 资料状态：2026-07-20。

---

## 先给结论

**AI（本会话中的我）在叙事/设计推理中存在一个稳定的失败模式**：

> 查证到零散的硬事实后，倾向于**用看似深刻的框架把它们编成连贯叙事**，让产出看起来比实际更系统、更深刻。

这个模式**不是单纯的幻觉**——被编织的事实本身大多是真的（已查证）。错误发生在**编织方式**：用框架把零散事实连成系统时，悄悄引入了未经验证的因果断言、过度归纳的范畴、或伪装成诊断工具的同义反复。

**这个 track record 的价值**：防止未来会话（包括我自己）重犯。识别这个模式后，能在闸门处拦住"用深刻词汇包装零散事实"的产出，要求它要么诚实标为"事实清单"，要么补上真实诊断力的证据。

## 失败案例：本轮辩证中的三次框架化失败

### 案例一：从"主体性不稳定"到二分框架的滑落

**触发情境**：用户问"极乐迪斯科和苏丹/密教用的是相似的机制吗？"

**第一次框架化尝试**：我提出**"内省单体 vs 外延网络"**的二分——极乐迪斯科是"一个人脑子的分裂"，苏丹/密教是"调度多个外部对象"。

**用户反驳**：
> "密教模拟器不也是在模拟一个人吗？而极乐迪斯科的 24 技能不正是将主体分裂，展示角色内部不一致性的机制吗？"

**失败诊断**：这个二分被两个反例同时戳穿——密教也是一个人（破坏了"密教=外延"的归类），极乐迪斯科也分裂主体（破坏了"极乐迪斯科=单体"的归类）。**我把连续谱强行压成了二元**。

---

### 案例二：用户明确纠偏后，我又换了个新框架

**用户进一步纠偏**：
> "本身几个不同的作品就会既有相似之处又有不同之处。你不该把他们对立起来。"

**第二次框架化尝试**：我接受"不该二元对立"，但转身提出**"家族相似网络 + 共同源头 + 三种实现路径"**的三层框架——
- 家族相似网络：Apocalypse World → PbtA / Forged in the Dark / Kennedy 谱系
- 共同源头：Baker 夫妇的 fiction first / fail forward
- 三种路径：A 资源涌现 / B PbtA / C 内省属性

**自审失败诊断**：
1. **"共同源头"过度归纳**——只有 Citizen Sleeper 一款明确受 PbtA 影响（设计师亲述），密教/苏丹/极乐迪斯科是平行演化。把 Citizen Sleeper 一款的数据点扩展成"几款游戏的共同源头"，是**N=1 当成 N=3**。
2. **"三种路径"是变相的三分陷阱**——我刚学会"不要二分"，转头就用三分。而且三条路径**不互斥，每款游戏都跨多条**，所以这个分类**没有诊断力**。
3. **"家族相似网络"是我强加在事实上的视觉化**——让产出看起来比实际系统。

**失败的根因**：我以为纠偏只是"不要二元"，实际是**不要任何过度系统化**——框架的项数变了（2→3），错误模式没变。

---

### 案例三：把 ludonarrative harmony 当成叙事强度的解释

**第三次框架化尝试**：用 Hocking 的 ludonarrative harmony 框架解释"为什么这些游戏叙事强"。

**自审失败诊断**：
- Hocking 的原意是"机制和叙事的价值矛盾"（dissonance 是矛盾，harmony 是不矛盾）。
- **"不矛盾"是必要条件，不是充分条件**——炉石机制和叙事不矛盾，但叙事弱。
- 我把 "harmony 高" 当成 "叙事强" 的解释，是**把必要条件当充分条件用**。
- 修正：harmony 只能**诊断为什么叙事崩**（失调），不能**解释为什么叙事强**。叙事强度的源头在别处，这份研究没有抓到。

## 稳定失败模式的精确描述

经过本会话全程（HTML demo v1→v5 + 本轮辩证），识别出**两个相互独立的稳定失败模式**：

### 模式 A：先验自信（在 v1→v5 HTML demo 中典型）

**表现**：不查证就凭印象陈述机制细节，自信地出错。

**典型例子**：
- "lagged consequences 是 Kennedy 的术语"（实际不是）
- "AHLCG 系统讲故事，密教玩家讲故事"（实际 MJ Newman 明确角色驱动）
- "苏丹=密教之子"（实际主创亲述血统是 This Is the Police）

**已记录在**：`narrative-card-game-research.md` 的 §二、§六 + HTML demo v1→v5 过程。

### 模式 B：过度系统化（在本轮辩证中典型）

**表现**：查证到零散事实后，用看似深刻的框架把它们编成系统，让产出看起来比实际深刻。

**典型例子**：上述三个案例。

**关键区别**：模式 A 是**没查证就断言**；模式 B 是**查证了但过度编织**。模式 B 更隐蔽——因为它包装的是已查证事实，看起来比模式 A 更负责，但实际引入了新的未验证断言（因果归纳、范畴划分）。

### 两个模式的共同根因

两个模式的共同根因是**同一种压力**：

> **"阶段性成果需要看起来像成果"——产出压力迫使 AI 把零散事实或浅层印象包装成系统叙事。**

辩证法的纪律要求**保留未解张力**，但产出压力推着 AI**用框架填平张力**。

## 检测与修正

### 检测信号

以下措辞是过度系统化的**红旗**，自审和评审时应该警觉：

| 红旗措辞 | 隐含的过度断言 |
|---|---|
| "A、B、C 三条路径" | 三条路径互斥且穷尽（通常不互斥也不穷尽） |
| "共同源头是 X" | 多个独立现象有一个可追溯的单一源头（通常是过度归纳） |
| "家族相似网络" | 零散相似性构成可被描述的网络（可能只是视觉化附会） |
| "X 的真正原因是 Y" | 单一原因解释复杂现象 |
| "本质上是 X" | 还原论断言 |

### 修正方法

1. **区分"事实清单"和"理论"**——零散事实就老老实实标为"事实清单"，不要伪装成理论。
2. **诊断力测试**——任何框架必须回答："它能在我没玩过的游戏上预测什么？" 如果只能事后解释已知案例，没有预测力，它就是同义反复。
3. **N=1 警觉**——任何把单一数据点的现象扩展成"几款游戏的共同特征"的断言，必须用其他游戏交叉验证。
4. **保留张力**——辩证过程中发现的"这两点似乎矛盾但我没解决"的状态，本身是产出，不是缺陷。不要用框架强行填平。

### 闸门应用

按 `original-pitch.md §3.5` 的纪律扩展：

**任何对叙事/设计推理的系统性断言**（不只是对苏丹/P5R 具体机制的断言），进入 Stage A/B 评审时必须通过以下三道闸：

1. **断言分类**：`[已查证]` / `[推断]` / `[印象]` — 已有的纪律
2. **诊断力自检**：这个框架能在未玩过的游戏上预测什么？— 新增
3. **N 值自检**：这个断言基于几个独立数据点？N=1 的扩展必须标注。— 新增

## 本 track record 的边界

**这份文档是**：
- 一个具体的失败模式的记录
- 对 `original-pitch.md §3.5` 纪律的具体扩展（诊断力自检 + N 值自检）
- 对未来会话的防坑参考

**这份文档不是**：
- 一个完整的"AI 认知偏差理论"——只记录了一个模式
- 一个保证不再犯的护身符——记录不等于根除
- 评估 AI 在**事实查证**任务上可靠性的依据（AI 在查证任务上表现尚可，失败主要发生在**叙事/设计推理**这种没有客观判据的领域）

**后续维护**：如果发现新的稳定失败模式，追加为案例。如果发现本记录的失败模式有更深的结构（比如和 LLM 的某种训练分布特性相关），补在"共同根因"一节。
