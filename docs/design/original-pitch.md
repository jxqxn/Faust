## 3.5 评审准入：断言分类与来源标注（2026-07-19 建立）

> 本节是 Stage A/B 评审的**前置纪律**：任何提交评审的提案，对两个母体（苏丹/P5R）的每条陈述必须自带分类标签，否则评审不予受理。

**为什么要这条**：2026-07-19 的 HTML 原型 track record（v1→v5 五轮纠错）暴露了一个稳定模式——AI 在不查资料时对苏丹/P5R 的理解会**自信地说错**，且"听起来像专家"和"真的懂"在机制细节上几乎不相关。calendar_coop 翻车（8 提交 / 34 文件 / 2263 行被 revert）就是"凭印象的断言没被识别为印象，直接进了实现"的结果。

**每条对苏丹/P5R 的陈述必须标注四类之一**（写在陈述末尾的方括号里）：

| 标签 | 含义 | 可信度 |
|---|---|---|
| `[已查证]` | 有可追溯来源：反编译指针（`[SRC: ...]`）、BWIKI/多源攻略交叉、Faust 工程测试覆盖 | 高，可作为提案依据 |
| `[推断]` | 基于已查证事实的合理推导，但没有直接来源 | 中，提案里需标注"待验证" |
| `[印象]` | AI 或人凭记忆/直觉说出，无来源 | 低，不能作为提案依据，只能作为待验证假设 |
| `[不确定]` | 显式承认不知道 | 诚实标注，比 `[印象]` 更可信赖 |

**操作规则**：
1. **含 `[印象]` 断言的提案不得进入 Stage A/B 评审**。这些断言必须先升级到 `[已查证]` 或 `[推断]`（通过查证流程），或降级为 §5 创意素材区的开放假设。
2. **`[推断]` 断言数量超过提案总断言 1/3 的提案**，评审必须附"待验证清单"，列出每个 `[推断]` 升级到 `[已查证]` 所需的查证路径。
3. **提案者不能给自己的断言标 `[已查证]` 而不提供来源**。`[已查证]` 必须配来源指针（反编译路径 / 攻略 URL / 测试文件:行号）。
4. **评审会议上，任何母体相关断言若被质疑，提案者必须能在 5 分钟内拿出来源或承认降级**。拿不出又不降级的，断言作废。
5. **系统性/框架性断言必须通过诊断力自检**（2026-07-20 追加）：任何"X 是 Y 的本质/源头/共同点"式的系统性断言，必须回答"它能在未玩过/未分析过的游戏上预测什么？" 只能事后解释已知案例、没有预测力的框架，标为同义反复，不能作为提案依据。**这条针对"用深刻词汇包装零散事实"的失败模式**，详见 `docs/research/ai-over-systematization-track-record.md`。
6. **跨游戏/跨作品的归纳断言必须标注 N 值**（2026-07-20 追加）：把某一作品的现象扩展为"几款游戏的共同特征"时，必须说明该断言基于几个独立数据点。N=1 的扩展必须显式标注，评审时按单点证据处理，不能当成跨作品共识。

**对苏丹的查证路径**（按可信度排序）：
1. `Faust-local-source/_unpack/` 的 `.c` 反编译 + `dump.cs`（事实本身，最高权威）
2. Faust 工程里带 `[SRC: ...]` 指针的代码 + 测试覆盖
3. BWIKI 词条（多源交叉时可信）
4. 知乎/游民/NGA 等攻略（单源时仅作线索）

**对 P5R 的查证路径**：没有反编译，只能靠玩家 wiki + 设计师访谈 + 你自己的专家判断。**这意味着 P5R 相关断言天然比苏丹弱一档**——任何 P5R 融合提案都要诚实标注这个不对称。

**这条纪律的元目的**：把 v1→v5 track record 暴露的"自信 ≠ 准确"问题（**模式 A：先验自信**），以及 2026-07-20 第二轮辩证暴露的"过度系统化"问题（**模式 B：用框架包装零散事实**），从"靠提案者自觉"升级为"靠评审流程强制"。两个模式的完整记录在 `docs/research/ai-over-systematization-track-record.md`。不是要求所有断言都 `[已查证]`，是要求**断言的置信度是显式的，且系统性断言的诊断力和样本量也是显式的**，让评审能在已知风险下决策。

#### 5.1.3 P5R 特长拆解：可提炼的体验原子 vs 不能搬的形式

**✅ 可提炼的体验原子**（可作为创意素材，是否采用待定）：

- 时间有限性：今天给了谁/什么，另一件事就错过或变化。苏丹有期限，但没有"日常事务争夺同一时段"。
- 关系经营改写行动能力：与某人变熟后，解决压力问题的方式真实地不同了。
- 人物有时间窗口：不是随时能见到，会缺席、会出现在特定场合。
- 压力 vs 成长争夺同一份时间预算。

**❌ 不能搬的形式/题材**（搬了就是换皮，calendar_coop 的错误）：

- after_school / night / 昼夜时段结构
- palace / mementos / 殿堂 / 印象空间
- calling_card / treasure_secured / 预告信 / 秘宝
- Confidant / Rank / 隐藏 CP / affinity 数值升级
- arcana / persona / 塔罗 / 面具
- 5 阶段线性案件流程

#### 5.1.4 融合方向假设（待评审）

真正的融合（如果走这个方向）不应该是独立模式，而是**强化 Faust 已有人物卡的关系维度**。Faust 已经有：苏丹卡（外部压力）+ 仪式（决策容器）+ 事件（叙事）+ 人物卡（状态载体）。苏丹原作里人物是"能力集合+状态载体"，但关系是静态的。

假设方向（**非结论**）：在 Faust 的桌面上，让人物卡之间形成**有时间窗口的关系网络**——经营这些关系会改变玩家解决苏丹卡的方式。例如：

- 某个人物只在特定日期 / 特定条件下出现在桌面；
- 关系深度解锁新的仪式槽位解法 / 新的掉落路线 / 新的事件分支；
- 忽视一段关系会让某条解题路线消失或变贵；
- 关系状态（承诺、缺席、亏欠）被后续事件读取。

P5R 的特长通过苏丹已有的卡牌语言表达，而不是另起炉灶。
